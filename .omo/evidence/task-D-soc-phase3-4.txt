DESIGN SUMMARY:
  - Module: doorbell
  - Interface: AMBA APB v2.0 slave at base 0x4000_5000 (decoded by apb_decoder)
  - Registers (4 × 32-bit, offset within 4 KB window):
      0x00  HOST_TAIL  (RW)  Host writes after enqueuing commands
      0x04  NPU_HEAD   (RW)  NPU firmware writes after consuming commands
      0x08  HOST_HEAD  (RW)  Host completion ring head pointer
      0x0C  NPU_TAIL   (RW)  NPU completion ring tail pointer
  - Interrupt:  doorbell_irq = (HOST_TAIL != NPU_HEAD)  (combinational)
  - Address validation: only 0x00~0x0F are valid; writes outside this range
    are silently ignored, reads return 0 (prevents aliasing).

FIRMWARE COMPATIBILITY:
  - npu_firmware.c main loop (lines 294-324):
      Reads  HOST_TAIL, NPU_HEAD → checks (host_tail == npu_head) for WFI
      Writes NPU_HEAD after consuming commands
      doorbell_irq auto-clears when HOST_TAIL == NPU_HEAD
  - npu-regmap.h struct npu_doorbell_t (lines 99-106): register layout matches.

KEY DESIGN DECISIONS:
  1. All 4 registers are RW at RTL level. The npu_firmware.c writes NPU_HEAD
     (line 323: NPU_DB->NPU_HEAD = npu_head), so NPU_HEAD must be writable.
     Access control (who writes what) is at the protocol/firmware level.
  2. doorbell_irq is purely combinational from the two registers — no pipeline
     delay, no edge detection. Firmware polling loop reads registers directly
     for the WFI condition; the IRQ is a wake-up signal.
  3. Undefined addresses (paddr[11:4] != 0) return 0 and ignore writes.
  4. pready = psel && penable (zero wait states). pslverr = 0 always.
