# 架构选型报告 B — 3D DRAM 高带宽 NPU（修订版）

**日期**: 2026-06-29  
**需求**: 高端端侧 / 轻量加速卡，3D DRAM 堆叠，跑 7B 模型，目标 ≥150 TPS  
**模型**: Qwen2.5-7B INT4 (3.5 GB)，28 层, hidden=3584  
**内存**: 3D DRAM 1024 GB/s (1 TB/s)，可用 870 GB/s

---

## 0. 场景本质：瓶颈从带宽转移到 PE 结构

```
LPDDR5 (51.2 GB/s): 带宽瓶颈 → 所有引擎同吞吐 → 选面积最小的 (FSA)
3D DRAM (1 TB/s):   PE瓶颈   → 引擎计算效率决定吞吐 → 选per-tile成本最低的
```

**7B INT4 理论天花板**: 1024 GB/s ÷ 3.5 GB ≈ **293 tok/s**  
**80% BW 利用率**: ~234 tok/s  
**INT8 权重天花板**: 1024/7 ≈ 146 tok/s（若需 INT8 精度）

---

## 1. PE 结构对比：为什么 block 在高带宽下碾压 FSA

### 1.1 Per-tile 成本差异（根本原因）

| PE 结构 | per-tile 公式 | W=256 | W=1024 | 特性 |
|------|------|:---:|:---:|------|
| **block** | **H + 4** | 132 | **132** | 与 W 无关，全并行广播 |
| FSA | H + M + W | 385 | **1153** | 含脉动管线排空（W cycles） |
| GMMA | H + M + W | 385 | 1153 | 同 FSA，TMA 只省 DMA |

### 1.2 完整 7B 模型 TPS 对比

| W | N_tiles | FSA TPS | block TPS | GMMA TPS | block/FSA |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 256 | 74 | 13 | 38 | 13 | **2.9×** |
| 512 | 37 | 16 | 76 | 16 | **4.8×** |
| 1024 | 19 | 16 | 138 | 16 | **8.7×** |
| 1536 | 13 | 15 | **174** | 15 | **11.3×** |

**FSA 在 W=1024 已达饱和（16 TPS）**：W 增大 → per-tile 管线成本（W cycles）同步增长 → 抵消了 N_tiles 减少的收益。**FSA 存在最优 W，超过后性能反降。**

**block 无此限制**：per-tile 成本恒定 132 cycles。W 越大 → N_tiles 越少 → TPS 线性增长，直到触达带宽天花板 (~234 tok/s)。

### 1.3 物理直觉

```
FSA (脉动阵列):
  激活 ──→ [PE0]→[PE1]→...→[PE255]  ← 必须等 W 个周期排空
  每 tile 成本 = 128(填) + 1(算) + 256(排空) = 385 cycles

block (全并行广播):
  激活 ──┬→ [PE0]
         ├→ [PE1]   ← 同时到达，无管线延迟
         └→ [PE255]
  每 tile 成本 = 128(K归约) + 4(开销) = 132 cycles
```

## 2. 达标分析：150 TPS 需要什么

### 2.1 150 TPS 的瓶颈分解

目标：150 TPS → 每 token ≤ 6.67ms → 每层 ≤ 238K cycles

**FFN_gate 是最大瓶颈**（3 个 FFN 占总计算量的 87%）：

```
FFN_gate cycles = ceil(K/H) × ceil(N/W) × (H + overhead)
                = ceil(3584/H) × ceil(18944/W) × (H + 4)   [block]
```

### 2.2 阵列方案

| 阵列 | K_tiles | N_tiles | TPS | PE 面积 | 总面积* |
|------|:---:|:---:|:---:|:---:|:---:|
| block 128×256 | 28 | 74 | 38 | 64mm² | **113mm²** |
| block 128×512 | 28 | 37 | 76 | 128mm² | 177mm² |
| block 128×1024 | 28 | 19 | 138 | 256mm² | 305mm² |
| **block 128×1536** ✓ | 28 | 13 | **174** | 384mm² | **433mm²** |
| FSA 128×1024 | 28 | 19 | 16 | — | 不可行 |

> \*面积 = PE + 固定(25) + SRAM(12) + PHY(12)。3D DRAM PHY 按 HBM 级别 µbump 密度估算 ~12mm²/1024-bit，非传统 DDR PHY 线性缩放。

### 2.3 推荐配置

| 参数 | 值 | 依据 |
|------|------|------|
| **引擎** | **block 128×1536** | per-tile 恒定 132cycles，W 扩展性好 |
| **SRAM** | **8 MB** | KV cache 缓冲（7B 每层 ~2MB） |
| **精度** | INT4 权重 / INT8 激活 | 精度通过 |
| **内存** | 3D DRAM 1024 GB/s | 1 TB/s |
| **阵列** | 128 行 × 1536 列 | 196K PEs, N_tiles=13 |
| **面积** | **~433 mm²** | PE 384 + 固定 49 |
| **功耗** | ~150W（估） | 196K PEs @ 1GHz |

### 预期性能

| 负载 | 性能 |
|------|:---:|
| Qwen2.5-7B decode | **174 tok/s** |
| Qwen2.5-7B TTFT (128 tok) | **~15 ms** |
| Qwen2.5-3B decode | ~800 tok/s |
| 等效生成速度 | ~10,000 字/分钟 |

## 3. 两场景引擎选择（最终结论）

| | 场景 A: LPDDR5 + 3B | 场景 B: 3D DRAM + 7B |
|---|---|---|
| 瓶颈 | **DRAM 带宽** | **PE 阵列利用率** |
| 推荐引擎 | **FSA** | **block** |
| 选择逻辑 | 带宽瓶颈掩盖 PE 差异 → 选面积最小 | PE 结构决定扩展性 → 选 per-tile 成本最低 |
| 核心指标 | TPS/mm² (FSA 0.70 vs block 0.42) | TPS @ W scale (block 线性, FSA 饱和) |
| 面积 | 32mm² | 433mm² |
| TPS | 23 | 174 |

**DSE 价值**: 同一引擎库，通过切换内存参数和 PE 结构，自动选出最优架构。**不存在万能引擎**——脉动阵列在带宽受限时胜面积，全并行广播在带宽充裕时胜扩展性。

---

## 4. 模型说明与局限

1. **单 token decode 假设** — 无 batch。多用户并发时 FSA 的 pipeline 利用率会显著改善
2. **PE 利用率 0.6-0.8%** — M=1 decode 场景下 systolic/block 阵列的固有局限，真实硬件通过 batch 推理缓解
3. **3D DRAM PHY 面积** — 按 HBM 级别 µbump 估算 ~12mm²/1024-bit，非传统 DDR PHY 线性缩放
4. **INT8 vs INT4** — INT8 权重天花板 146 TPS，如需 INT8 精度则 174 TPS 配置不可行，需更大带宽或 INT4
