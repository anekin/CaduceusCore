## S1 低成本端侧 NPU — LPDDR5 + 3B LLM

### 输入参数

| 参数 | 值 | 来源 |
|------|------|------|
| 模型 | Qwen2.5-3B INT4 (1.5 GB, 28层) | 显式指定 |
| 内存 | LPDDR5-6400 64-bit, 51.2 GB/s (有效 43.5 GB/s) | 显式指定 |
| seq_len | 128 | 显式指定 |
| TTFT 约束 | < 200ms | 显式指定 |
| TPS 约束 | ≥ 20 | 显式指定 |
| 面积约束 | < 80mm² | 显式指定 |
| 工艺 | 12nm TSMC | 显式指定 |

### Preflight 预判

| 指标 | 值 |
|------|------|
| BW 天花板 | 29 TPS (=43.5 GB/s ÷ 1.5 GB) |
| 算力天花板 | 500-10917 TPS (6-131 TOPS) |
| **瓶颈** | **BANDWIDTH** — 算力严重过剩 |
| TTFT 最少 TOPS | 3.8 (2×3B×128÷200ms) — 极低 |
| 组件清单 | DRAM PHY ✓ / PCIe ✓ / TSV ✗ |

### DSE 扫描结果

|--------|:---:|:---:|:---:|:---:|:---:|:---:|
| **FSA** | **128×256** | **33** | **68mm²** | **23** | 88ms | 0.34 |

| FSA | 192×128 | 25 | 65mm² | 23 | 94ms | 0.35 |

| block | 128×256 | 33 | 78mm² | 23 | 45ms | 0.29 |

### 敏感性分析

| 参数 | ΔTPS | ΔArea | 结论 |
|------|:---:|:---:|------|
| SRAM | 8.6% | 14.5% | 4MB 最优，边际收益递减 |
| MACs | 0.5% | 19.8% | 算力无关 — BW 瓶颈 |
| Engine | 0.9% | 5.7% | FSA vs block 差异小 |

### 推荐配置

| 参数 | 值 |
|------|------|
| 引擎 | **FSA 128×256** |
| TOPS INT8 | 33 |
| SRAM | 4 MB |
| 精度 | INT4 权重 / INT8 激活 |
| 内存 | LPDDR5-6400 64-bit |
| 面积 @ 12nm | **~68 mm²** |
| TPS | 23 |
| TTFT (128) | 88 ms ✓ |
| 功耗 | ~14W（估） |

**选择逻辑**: LPDDR5 BW 瓶颈，所有配置 TPS 集中在 21-23。33 TOPS 选 FSA 而非 block——面积少 10mm²，TTFT 余量足够，且 S2/S3 已用 block，差异化利于 portfolio。

### 敏感性分析

| 参数 | ΔTPS | ΔArea | 结论 |
|------|:---:|:---:|------|
| H | 18.2% | 92.2% | 主驱动 — TPS 在 H≥64 后饱和 (BW→engine 瓶颈) |
| SRAM | 4.3% | 13.7% | 低影响，512KB 足够 |

### 推荐配置

| 参数 | 值 |
|------|------|
| 引擎 | **block 80×1536** |
| TOPS INT8 | 123 |
| SRAM | 512 KB |
| 精度 | INT4 权重 / INT8 激活 |
| 内存 | On-chip 3D DRAM 5GB, BW ≈ 930 GB/s |
| 面积 @ 12nm | **~124 mm²** |
| TPS | 197 |
| TTFT (1024) | 65 ms ✓ |
| 功耗 | ~25W（估） |

**选择逻辑**: H=80 是同时满足 >100 TOPS 和 TTFT<200ms 的最小面积。TPS 在 H≥64 后饱和（从 engine 瓶颈转为 BW 瓶颈），继续增大阵列对 decode 无益。SRAM 512KB 充足。

## S3 具身智能 NPU — 3D DRAM + VLM/VLA
## ═══════════════════════════════════

### 输入参数

| 参数 | 值 | 来源 |
|------|------|------|
| 模型 | Qwen2.5-7B INT4 (3.5 GB) + Qwen-VL ViT ~675M | 显式指定 |
| 视觉编码器 | ViT-Huge (1280-dim, 32层), 4-crop 动态分辨率 | 基于 Qwen2.5-VL 实测 |
| 视觉负载 | 2583 GMACs/帧 (5.2 TFLOPs) | trace 生成 |
| 内存 | On-chip 3D DRAM 5GB, BW=面积×7.5 GB/s/mm² | 显式指定 |
| seq_len | 1024 | VLM 长上下文 |
| TTFT 约束 | < 200ms | 显式指定 |
| **FPS 约束** | **≥ 10 FPS (pipeline)** | 显式指定 |
| 响应长度 | 10-20 tok (VLA 动作指令) | 推导 |
| 工艺 | 12nm TSMC | 显式指定 |

### Pipeline 模型

```
Pipeline FPS = 1000 / max(vision + prefill, decode_N_tokens)
```

Vision (Qwen-VL ViT 4-crop) ∝ 1/TOPS  
Prefill (1024 tok) ∝ 1/H  
Decode (N tok) = N / TPS

### DSE 扫描结果 (10 FPS 评估)

| H×W | TOPS | 面积 | BW | TPS | Vis | Prefill | 10tok FPS | 20tok FPS |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 32×1536 | 49 | 66mm² | 494 | 146 | 65ms | 160ms | 4.4 ✗ | 4.4 ✗ |
| 64×1536 | 98 | 105mm² | 784 | 197 | 34ms | 80ms | 8.8 ✗ | 8.8 ✗ |
| **80×1536** | **123** | **124mm²** | **930** | 197 | 26ms | 65ms | **11.0 ✓** | 9.9 (边界) |
| 96×1536 | 147 | 143mm² | 1076 | 198 | 22ms | 55ms | 13.0 ✓ | 9.9 (边界) |
| 128×1536 | 197 | 182mm² | 1366 | 198 | 18ms | 40ms | 17.3 ✓ | 9.9 (边界) |

> 注：20 tok 时 decode 时间 = 20/198 = 101ms，刚好超过 100ms 阈值，导致 FPS 跌至 9.9。

### 推荐配置

| 参数 | 值 |
|------|------|
| 引擎 | **block 80×1536** |
| TOPS INT8 | 123 |
| SRAM | 512 KB |
| 内存 | On-chip 3D DRAM 5GB, BW ≈ 930 GB/s |
| 面积 @ 12nm | **~124 mm²** |
| Vision (4-crop) | 26 ms |
| Prefill (1024) | 65 ms |
| Decode (10 tok) | 51 ms |
| **Pipeline FPS** | **11.0** ✓ |
| Diffusion SD 1.5 | ~12 img/s (LCM 4步) |

**选择逻辑**: 同 S2 芯片。H=80 是 10 FPS 的最小面积点。瓶颈在 vision+prefill (91ms)，而非 decode (51ms)。20 tok 时 decode 成为新瓶颈 (101ms)，刚好不及格——如需 20 tok @ 10 FPS 则选 H=96。
